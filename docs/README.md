# docs/ 配下の導入ドキュメントについて

このディレクトリには、ConsoNanceの導入・移行手順をまとめた静的ドキュメントを配置しています。

## 重視した点

### 1) CUIに慣れていない人でも迷わないこと
- 画面のスクリーンショットを中心に、手順を「何を見て」「何を入力して」「何が起きたら成功か」で追えるようにしました。
- `(y/n)` のような前提知識が必要な表記は、補足（折りたたみ）で意味を説明しています。

### 2) HTML + CSS のみで完結し、そのまま公開できること
- JavaScriptは使わず、静的ホスティング（例: S3）へそのまま配置して公開できる構成にしています。
- CSSは `docs/styles.css` に集約し、ページ間で見た目と読みやすさを統一しました。

### 3) 「新規導入」と「移行」を分けて、目的別に読めること
- 入口ページ（`index.html`）から、読者が自分の状況に合わせてすぐに目的のページへ遷移できるようにしました。
- 「新規導入」「移行」それぞれで、必要な情報の順序と粒度が異なるため、ページを分けています。

### 4) 画像を同梱し、リンク切れや外部依存を減らすこと
- ドキュメントで使用する画像は `docs/assets/` 配下に保存し、HTMLから相対パスで参照しています。
- これにより、公開先で外部URLに依存せず、表示の安定性を高めています。

## ページ構成
- `docs/index.html`: 入口（どれを見ればよいか、注意事項）
- `docs/setup.html`: 新規導入（Discord Bot作成〜招待〜初回起動〜操作）
- `docs/migrate.html`: 移行（Discord Audio Stream Botからの移行手順・トラブルシュート）
- `docs/styles.css`: 共通スタイル

## 画像・素材の配置
- `docs/assets/`: ドキュメント本体で参照する画像
  - `add-bot-01.png` / `add-bot-02.png` など、手順の説明に対応
  - `github-download.png` など、ダウンロード手順の説明に対応
- `docs/assets/blog/`: 追加で使用する画像（必要に応じて参照）

## 更新時のガイド
- **画像を差し替える場合**: `docs/assets/` に保存し、HTMLの `<img src="assets/...">` を更新してください。
- **文章を更新する場合**: まず「手順の順番」を崩さず、画像の順序と整合しているかを確認してください。
- **リンクを追加する場合**: 外部リンクは `target="_blank" rel="noopener"` を付けています（既存の書き方に合わせてください）。
